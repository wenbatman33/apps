# 🎲 Monopoly Party — 手機直屏大富翁遊戲開發規格書

> **目標平台**：iOS / Android（直屏 Portrait 9:16）
> **目標受眾**：5～16 歲兒童與青少年
> **設計意象**：任天堂風格 — 可愛、乾淨、純色系、圓潤造型
> **技術方案**：遊戲引擎（Phaser 3 / PixiJS + 物理引擎）或 Unity WebGL
> **品質要求**：精緻度等同商業手遊，不接受粗糙 placeholder 風格

---

## 一、核心設計哲學

### 1.1 視覺風格定義

```
風格關鍵字：Nintendo-like / Kawaii / Flat-3D / Pastel / Rounded
```

- **色彩系統**：以柔和純色為主調（馬卡龍色系），避免漸層過度使用
  - 主色：柔和藍 `#5B9BD5`、薄荷綠 `#7ED6A0`、暖橘 `#F5A623`
  - 強調色：櫻花粉 `#FFB7C5`、淡紫 `#B8A9E8`、檸檬黃 `#FFE066`
  - 中性色：奶白 `#FFF8F0`、淺灰 `#E8E4E0`、深咖 `#4A3728`
  - 危險/警告：珊瑚紅 `#FF6B6B`
  - UI 背景：使用半透明毛玻璃效果 `rgba(255, 255, 255, 0.85)` + `backdrop-filter: blur(12px)`

- **圓角規範**：所有 UI 元素最小圓角 12px，按鈕 16px，卡片 20px，彈窗 24px
- **陰影規範**：使用柔和投影 `0 4px 16px rgba(0,0,0,0.08)`，禁止硬陰影
- **字體**：
  - 標題：圓體粗體（如 Rounded M+ Bold / Nunito Bold）
  - 內文：圓體中等（Nunito SemiBold）
  - 數字：等寬圓體（Nunito Mono 或自訂像素字體用於金幣顯示）

### 1.2 角色與吉祥物設計

- 4～6 個可選角色，風格類似「動物森友會」的 Q 版動物角色
- 每個角色有 **待機動畫**（呼吸感晃動）、**行走動畫**（彈跳式移動）、**歡呼動畫**（獲得獎勵時）、**沮喪動畫**（被攻擊時）
- 角色比例：大頭 2.5 頭身，眼睛佔臉部 40%
- 推薦角色陣容：
  1. 🐱 小橘貓（預設）— 活潑型
  2. 🐶 柴犬 — 忠誠型
  3. 🐰 垂耳兔 — 可愛型
  4. 🦊 小狐狸 — 聰明型
  5. 🐻 小棕熊 — 力量型
  6. 🐧 企鵝 — 冷酷型（隱藏角色，通關後解鎖）

### 1.3 音效與音樂方向

- **BGM 風格**：輕快爵士 / 可愛電子音樂，類似《動物森友會》《星之卡比》
- **音效設計**：
  - 骰子滾動：木質骰子在桌面彈跳的清脆聲
  - 角色移動：Q 彈的「噗噗噗」腳步聲
  - 金幣收集：清脆的「叮～」聯鎖音效（連續收集時音階上升）
  - 建造完成：歡樂的號角短音 + 煙花碎紙音效
  - 按鈕觸碰：柔軟的「咕」聲
  - 負面事件：可愛的「哇哦～」下降音效（不恐怖）

## 三、核心遊戲機制

### 3.1 骰子系統

```javascript
// 骰子核心邏輯
DiceSystem {
  // 基礎機制
  diceCount: 2,                    // 雙骰
  range: [1, 6],                   // 每顆 1~6
  totalRange: [2, 12],             // 總步數
  
  // 倍率系統
  multiplierOptions: [1, 2, 5, 10],
  currentMultiplier: 1,            // 影響金幣收益，同時消耗對應倍數骰子
  
  // 骰子回復
  maxDice: 200,                    // 初始上限
  regenRate: 5,                    // 每小時自然回復
  regenCap: 50,                    // 自然回復上限
  
  // 特殊規則
  doublesBonus: true,              // 骰子相同 = 額外一次機會
  criticalRoll: {                  // 暴擊機制
    chance: 0.05,                  // 5% 觸發
    multiplier: 3,                 // 收益 x3
    animation: "rainbow_dice"      // 彩虹骰子動畫
  }
}
```

**骰子動畫流程**：
1. 玩家按下擲骰按鈕 → 按鈕下壓動畫 + 震動反饋
2. 骰子從畫面底部彈出 → 物理模擬旋轉（帶彈跳）
3. 骰子停止 → 數字放大閃光 → 如果是暴擊，全螢幕彩虹特效
4. 角色 Token 開始逐格移動 → 每格有踩踏特效
5. 到達目的格 → 觸發對應事件

### 3.2 棋盤格子類型

| 格子類型 | 圖示 | 觸發效果 | 出現頻率 |
|---------|------|---------|---------|
| 💰 金幣格 | 金幣堆 | 獲得金幣 × 倍率 | 35% |
| 🏠 物業格 | 彩色房屋 | 進入建造/升級介面 | 20% |
| 💎 寶箱格 | 寶箱 | 隨機獎勵（骰子/金幣/道具） | 10% |
| ❓ 機會格 | 問號卡片 | 抽取機會卡（正面/負面事件） | 15% |
| 🛡️ 防護格 | 盾牌 | 獲得 1 個防護盾 | 5% |
| 👊 突襲格 | 紅色拳頭 | 突襲其他玩家的金庫 | 10% |
| ⭐ 特殊格 | 星星 | 觸發當前活動事件 | 5% |

### 3.3 建築系統（地標升級）

每個主題地圖有 **5 個地標建築**，每個地標有 **5 級升級**：

```
地標升級結構：
Level 1 → 空地 + 小帳篷        （花費: 100 金幣）
Level 2 → 小木屋               （花費: 500 金幣）
Level 3 → 磚造房屋             （花費: 2,000 金幣）
Level 4 → 豪華建築             （花費: 8,000 金幣）
Level 5 → 黃金地標 ✨          （花費: 25,000 金幣）

完成所有 5 個地標 = 解鎖下一張地圖主題
```

**建築動畫**：
- 升級時播放 2 秒建造動畫（腳手架搭建 → 建築成型 → 煙花慶祝）
- 每個地標有專屬的閒置動畫（煙囪冒煙、旗幟飄動、燈光閃爍）
- 完成全部地標時，播放 3 秒全城歡慶動畫 + 轉場到新地圖

### 3.4 社交攻防機制

#### 突襲（Raid）
- 踩到突襲格 → 顯示 3 個金庫（屬於隨機玩家）
- 選擇一個金庫 → 播放挖掘動畫 → 獲得對方部分金幣
- 被突襲的玩家收到通知 + 可發動「復仇」

#### 破壞（Shutdown）
- 特定機會卡觸發 → 選擇目標玩家的一個建築
- 播放「怪獸踩踏」動畫 → 建築降級 1 級
- 被破壞者的防護盾可抵擋一次

#### 防護盾系統
```
盾牌規則：
- 最多持有 3 個盾牌
- 每個盾牌抵擋 1 次攻擊（突襲或破壞）
- 通過踩防護格、每日登入、活動獎勵獲得
- 被攻擊時自動消耗，播放盾牌破碎動畫
```

## 四、介面設計規格

### 4.1 擲骰按鈕（核心互動元件）

```
尺寸：120 × 120 px（觸控熱區 140 × 140 px）
形狀：正圓形，帶 4px 白色描邊
底色：亮橘漸層 #F5A623 → #FF8C00
陰影：0 6px 20px rgba(245, 166, 35, 0.4)
圖示：白色骰子圖示居中

狀態動畫：
- idle：微幅上下浮動（2px，週期 2s）+ 柔和光暈脈動
- hover/press：縮小至 0.9 → 彈回 1.05 → 回 1.0（彈性緩動）
- disabled：灰階 + 透明度 0.5，顯示計時器
- rolling：旋轉 + 震動
```

### 4.2 倍率選擇器

```
佈局：水平排列 4 個膠囊按鈕
樣式：
  - 未選中：白底 + 灰色文字 + 1px 灰色邊框
  - 選中：主題色底 + 白色文字 + 微彈跳動畫
  - 骰子不足：紅色文字 + 搖晃提示

動畫：切換倍率時，骰子按鈕上的消耗數字做 flip 動畫
```

### 4.3 獎勵彈窗

```
結構：
┌────────────────────────┐
│    ✨ 恭喜獲得！ ✨      │  ← 標題（帶星星粒子）
│                        │
│    🎁                  │  ← 獎勵圖示（放大彈入動畫）
│   +2,500 金幣          │  ← 數字滾動計數動畫
│                        │
│   [    領取 ✓    ]     │  ← 主按鈕（綠色，帶光澤動畫）
│   [ 觀看廣告 x2 📺 ]   │  ← 次按鈕（金色描邊）
└────────────────────────┘

進場動畫：背景模糊 → 彈窗從中心彈性縮放出現（overshoot）
離場動畫：向上飄走 + 淡出
```

### 4.4 底部導航列

```
高度：70px + Safe Area 底部間距
背景：白色毛玻璃 blur(20px)
陰影：0 -2px 12px rgba(0,0,0,0.06)

每個 Tab：
  - 圖示 28×28px + 文字標籤 10px
  - 未選中：灰色 #999
  - 選中：主題色 + 圖示微彈跳 + 底部圓點指示器
  - 有通知時：右上角紅點脈動動畫
```

## 五、遊戲引擎技術規格

### 5.1 渲染架構

```
推薦引擎選擇（依優先順序）：

方案 A — Phaser 3 + Spine 動畫
  優點：Web 原生、輕量、社群大
  適合：快速原型 → 生產級 PWA
  渲染：WebGL 2.0（fallback Canvas 2D）
  動畫骨骼：Spine / DragonBones

方案 B — PixiJS 7 + Matter.js
  優點：極致渲染效能、靈活組合
  適合：需要大量粒子特效的場景
  物理：Matter.js（骰子物理模擬）

方案 C — Unity WebGL / Cocos Creator
  優點：完整 3D 支持、資源成熟
  適合：需要真 3D 城市場景
  注意：包體積較大，首次載入較慢
```

### 5.2 效能目標

```
目標幀率：60fps（最低 30fps）
目標裝置：iPhone SE 2 / Android 中階機（4GB RAM）
首次載入：< 5 秒（核心資源）
記憶體上限：< 300MB
素材格式：
  - 圖片：WebP（fallback PNG），使用 TextureAtlas 合圖
  - 音效：AAC / OGG（< 100KB/個）
  - 動畫：Spine JSON + Atlas
  - 字體：WOFF2 子集化
```
